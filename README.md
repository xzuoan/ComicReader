# 问题
> 内存占用问题，资源读取文件方法是通过Stream方式整体读取，遇到较大文件时耗费时间过长，且读取的压缩文件是乱序的，
> 目前方法是获取到全部数据流后放到列表中再排序，首页展示内容时只取列表第一条做封面，文件较多时同样耗时过长，且造成资源浪费，
> 应该有优化的空间。
